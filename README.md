# 📊 Data Analyst — Prompt Author Workbench

An end-to-end workbench for **data analysts** working as **prompt authors** — build, test, evaluate, analyze, and improve LLM prompts with rich data analytics, quality dashboards, and failure-pattern visualizations.

![Skills](https://img.shields.io/badge/Skills-Problem%20Solving%20%2B%20Data%20Analysis-7c3aed) ![License](https://img.shields.io/badge/license-MIT-blue)

## 🚀 Live Demo
Deploy `index.html` + `app.js` + `styles.css` to **GitHub Pages** — fully interactive in browser.

## ✨ Features
- ✍️ **Prompt Authoring** — template editor with `{variable}` detection
- 🚀 **Live Testing** — run prompts and validate against expected outputs
- 📈 **Analytics Dashboard** — quality trend, per-category, failure patterns, latency distribution (Chart.js)
- 📋 **Prompt Library** — search, filter, version, save, reuse prompts
- 📊 **Quality Scoring** — color-coded quality pills (high / med / low)
- 💾 **Persistent storage** via localStorage

## 🎯 Skills Demonstrated
- 🧠 Problem Solving
- 📊 Data Analysis (SQL · Pandas · statistical evaluation)
- ✍️ Prompt Engineering
- 📈 Visualization (Chart.js · Tableau-style dashboards)
- 🧪 Evaluation Design (rubrics · golden datasets)
- 🔍 Failure Analysis (error taxonomy · root-cause clustering)

## 🛠️ Tech Stack
- **Frontend:** Vanilla JS + Chart.js 4
- **Data:** localStorage (can plug into Snowflake / BigQuery / Postgres)
- **LLM:** simulated; plug into OpenAI / Azure OpenAI API

## 📦 Files
- `index.html`, `app.js`, `styles.css`
- `README.md`

## ⚙️ Run Locally
Open `index.html` in any modern browser. No build step.

## 🌐 Deployment
**GitHub Pages (free, instant):**
1. Push to a GitHub repo
2. Settings → Pages → Source: `main` / root → Save
3. Live at `https://<user>.github.io/<repo>/`

Also works on **Netlify, Vercel, Cloudflare Pages** — drag-drop deploy.
